# Git cheatsheet

Короткая шпаргалка по основным командам Git для ежедневной работы.

## Проверка состояния

```bash
git status
```

Показывает текущую ветку, измененные файлы и что уже добавлено в коммит.

```bash
git log --oneline --graph --decorate --all
```

Краткая история коммитов с ветками.

```bash
git diff
```

Показать изменения, которые еще не добавлены в коммит.

```bash
git diff --staged
```

Показать изменения, которые уже добавлены через `git add`.

## Обычный цикл работы

```bash
git status
git add .
git commit -m "Короткое описание изменений"
git push
```

Если пушишь новую ветку впервые:

```bash
git push -u origin имя-ветки
```

## Добавление файлов в коммит

Добавить все изменения:

```bash
git add .
```

Добавить конкретный файл:

```bash
git add путь/к/файлу
```

Убрать файл из будущего коммита, но оставить изменения в файле:

```bash
git restore --staged путь/к/файлу
```

## Коммиты

Создать коммит:

```bash
git commit -m "Описание изменений"
```

Изменить сообщение последнего коммита:

```bash
git commit --amend -m "Новое описание"
```

Добавить забытые изменения в последний коммит:

```bash
git add .
git commit --amend --no-edit
```

Если коммит уже был запушен, после `amend` обычно нужен:

```bash
git push --force-with-lease
```

## Ветки

Показать текущую ветку:

```bash
git branch --show-current
```

Показать локальные ветки:

```bash
git branch
```

Показать все ветки, включая удаленные:

```bash
git branch -a
```

Создать новую ветку:

```bash
git branch имя-ветки
```

Создать ветку и сразу переключиться на нее:

```bash
git switch -c имя-ветки
```

Переключиться на существующую ветку:

```bash
git switch имя-ветки
```

Скачать удаленные ветки и обновить информацию о них:

```bash
git fetch
```

Переключиться на удаленную ветку:

```bash
git switch имя-ветки
```

Если Git сам не нашел удаленную ветку:

```bash
git switch -c имя-ветки origin/имя-ветки
```

Переименовать текущую ветку:

```bash
git branch -m новое-имя
```

Удалить локальную ветку:

```bash
git branch -d имя-ветки
```

Удалить локальную ветку принудительно:

```bash
git branch -D имя-ветки
```

Удалить ветку на GitHub/GitLab:

```bash
git push origin --delete имя-ветки
```

## Получение изменений

Скачать изменения без слияния:

```bash
git fetch
```

Подтянуть изменения в текущую ветку:

```bash
git pull
```

Подтянуть изменения аккуратнее, поверх удаленной ветки:

```bash
git pull --rebase
```

## Слияние веток

Перейти в ветку, куда нужно влить изменения:

```bash
git switch main
```

Влить другую ветку:

```bash
git merge имя-ветки
```

После решения конфликтов:

```bash
git add .
git commit
```

Отменить merge, если он еще не завершен:

```bash
git merge --abort
```

## Откат изменений

Отменить изменения в конкретном файле до последнего коммита:

```bash
git restore путь/к/файлу
```

Отменить все незакоммиченные изменения:

```bash
git restore .
```

Важно: эта команда удаляет локальные изменения в файлах.

Убрать файл из индекса, но оставить изменения:

```bash
git restore --staged путь/к/файлу
```

Откатить последний коммит, оставив изменения в файлах:

```bash
git reset --soft HEAD~1
```

Откатить последний коммит и убрать изменения из индекса, но оставить их в файлах:

```bash
git reset HEAD~1
```

Откатить последний коммит вместе с изменениями:

```bash
git reset --hard HEAD~1
```

Важно: `reset --hard` удаляет изменения. Используй осторожно.

Создать новый коммит, который отменяет старый:

```bash
git revert хеш-коммита
```

Это безопаснее для уже запушенных коммитов, потому что история не переписывается.

## Временное сохранение изменений

Спрятать текущие изменения:

```bash
git stash
```

Спрятать с подписью:

```bash
git stash push -m "Описание"
```

Показать список stash:

```bash
git stash list
```

Вернуть последний stash:

```bash
git stash pop
```

Вернуть stash, но оставить его в списке:

```bash
git stash apply
```

Удалить последний stash:

```bash
git stash drop
```

## Работа с удаленным репозиторием

Показать удаленные репозитории:

```bash
git remote -v
```

Добавить origin:

```bash
git remote add origin ссылка-на-репозиторий
```

Изменить ссылку origin:

```bash
git remote set-url origin новая-ссылка
```

Запушить текущую ветку:

```bash
git push
```

Запушить текущую ветку впервые:

```bash
git push -u origin имя-ветки
```

## Полезные проверки перед push

```bash
git status
git branch --show-current
git log --oneline -5
```

Если в проекте есть проверки, обычно стоит выполнить:

```bash
npm run lint
npm run build
```

## Частые ситуации

### Хочу начать новую задачу

```bash
git switch main
git pull --rebase
git switch -c feature/название-задачи
```

### Хочу сохранить работу и отправить на GitHub

```bash
git status
git add .
git commit -m "Описание изменений"
git push -u origin имя-ветки
```

### Хочу переключиться, но Git не дает из-за изменений

Вариант 1: закоммитить изменения.

```bash
git add .
git commit -m "WIP"
git switch другая-ветка
```

Вариант 2: временно спрятать изменения.

```bash
git stash
git switch другая-ветка
```

Потом вернуть:

```bash
git stash pop
```

### Хочу отменить все локальные изменения

```bash
git status
git restore .
```

Если есть новые неотслеживаемые файлы, сначала посмотри список:

```bash
git clean -n
```

Удалить неотслеживаемые файлы:

```bash
git clean -f
```

Важно: `git clean -f` удаляет файлы, которых нет в Git.

### Хочу вернуться к конкретному коммиту временно

```bash
git switch --detach хеш-коммита
```

Вернуться обратно:

```bash
git switch имя-ветки
```

### Хочу найти, кто менял строку

```bash
git blame путь/к/файлу
```

## Мини-словарь

- `working tree` - файлы проекта прямо сейчас.
- `staged` / `index` - изменения, добавленные через `git add` и готовые к коммиту.
- `commit` - сохраненная точка истории.
- `branch` - ветка разработки.
- `remote` - удаленный репозиторий, например GitHub.
- `origin` - стандартное имя основного удаленного репозитория.
- `HEAD` - текущий коммит, на котором ты находишься.
- `HEAD~1` - коммит на один шаг назад.
